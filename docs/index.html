Long-Story Agent

Collaborative RAG-Agent for Long-form Narrative Generation

xiao-zi-chen
Independent Research 2026
Code Workflow Evaluation BibTeX

Long-Story Agent organizes narrative generation as a closed loop: task analysis, global and chapter-level planning, multi-agent writing, retrieval augmentation, consistency checking, and iterative revision with dynamic memory write-back.

Abstract

Long-form generation often suffers from structural drift, character inconsistency, and topic deviation. We present Long-Story Agent, a collaborative RAG-agent system that modularizes planning, writing, retrieval, memory, evaluation, and revision. The system maintains a closed-loop process in which chapter summaries, consistency signals, and revision feedback are continuously written back to memory for the next iteration. This design improves controllability and traceability for long narrative generation while keeping the workflow reproducible through explicit run logs and configuration files.

Method

1. Planning Layer

Analyzer and Organizer transform prompts into global constraints and chapter-level execution plans.

2. Generation Layer

Writer cooperates with Plot, Character, and World agents to draft coherent chapter content.

3. Retrieval and Memory

Retriever and static knowledge modules inject context, while memory stores narrative state updates.

4. Evaluation and Revision

Evaluator and consistency checker detect drift, and Realtime Editor applies targeted corrections.

Automatic Evaluation and Correction

Automatic scoring tracks coherence, emotional progression, and overall quality over iterations. The consistency checker records contradiction points and writes structured feedback to the revision stage, enabling the next generation cycle to address high-priority narrative issues.

Quick Start

Install

pip install -r requirements.txt

Set API Keys

set DEEPSEEK_API_KEY=your_key
# or
set OPENAI_API_KEY=your_key
set CODEX_API_KEY=your_key

Run

python main.py

Logs and generated artifacts are stored in runs/. Core configuration lives in config.py and config.toml.
